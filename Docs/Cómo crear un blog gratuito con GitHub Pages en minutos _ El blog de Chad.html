Crea un blog gratuito con GitHub Pages en minutos.

14 de marzo de 2021 • Chad Baldwin

Compartir en:

Esta entrada del blog está pendiente de actualización, ¡pero necesito tu ayuda!

Hola, compañero viajero de internet. Cuando escribí esta publicación hace más de un año, no imaginaba que se volvería tan popular. El 99% del tráfico proviene de simples búsquedas en Google; muchas veces aparece por encima de las instrucciones de GitHub para GitHub Pages. De hecho, la mayor parte del tráfico a este sitio web proviene únicamente de esta publicación.

Originalmente escribí esta publicación para ayudar a otros miembros de la comunidad de SQL Server que querían iniciarse en el mundo de los blogs. Dediqué mucho tiempo a personalizar este blog para que fuera compatible con el resaltado de sintaxis del lenguaje SQL Server. Pero… con el tiempo, el público al que va dirigida esta publicación ha cambiado bastante.

Si has leído esta publicación y quieres ayudarme a mejorarla, por favor, deja un comentario abajo, envíame un correo electrónico o un tuit y dime cómo podría mejorarla para ti y para futuros lectores.

¡Gracias!

Descargo de responsabilidad: Si estás familiarizado con GitHub Pages, Jekyll, etc., es posible que notes que digo algunas cosas "incorrectas"... esto se debe a que intento simplificar las cosas. Puede que diga "tienes que hacerlo de esta manera", pero tú, experto en Jekyll, sabrás que eso no es cierto.

“¿Cómo puedo empezar a bloguear?”

“¿Cómo puedo crear un blog?”

“¿Qué plataforma de blogs debería usar?”

“¿Cómo puedo conseguir ese resaltado de código tan chulo en mi blog?”

Así estaba yo hace tres meses. Experimenté con varias plataformas de blogs y parecían estar bien, pero por alguna razón, ninguna hacía lo que yo quería. Finalmente, descubrí GitHub Pages . Es un servicio de GitHub que permite alojar un sitio web estático (sin código de backend) de forma gratuita .

GitHub Pages es genial, pero cuando empiezas a aprender "cómo bloguear usando Jekyll", la cosa se vuelve bastante abrumadora... Empiezas a aprender sobre Ruby y Jekyll y cómo hacer que funcionen en Windows (lo cual es un fastidio), aprendiendo lenguajes como YAML y Liquid... Se complica muy rápido.

Esto es lo que he hecho

He tomado este blog, he eliminado lo innecesario y lo he convertido en una plantilla que puedes usar. He conservado las funciones que considero útiles para un bloguero que quiera escribir sobre SQL Server. Por ejemplo, diseñé esta plantilla para que admita el resaltado de sintaxis específico de T-SQL usando el tema de SSMS.

Tu primer blog no tiene por qué ser una página web sofisticada y con diseño profesional. Solo necesitas buen contenido, y crearlo debería ser tan sencillo como hacer unos clics, quizás rellenar algunos datos personales, y listo.

Eso es lo que he intentado preparar para ti.

Empecemos

Paso 1: Crea una cuenta de GitHub.

Si no tienes una cuenta de GitHub, crea una; de lo contrario, inicia sesión.

Nota: Tu nombre de usuario formará parte de la URL de tu sitio web, así que tenlo en cuenta al elegirlo.

Paso 2 - Abre la plantilla que he creado para ti.

Después de iniciar sesión en tu cuenta de GitHub, abre este enlace y sigue las instrucciones del paso 3:

https://github.com/chadbaldwin/simple-blog-bootstrap/generate

Paso 3: Nombra y crea tu repositorio.

*Si no nombras correctamente tu repositorio, nada de esto funcionará.

Nombra tu repositorio de esta manera:{Your GitHub username}.github.io

Si tu nombre de usuario de GitHub es “chadbaldwin”, entonces nómbralochadbaldwin.github.io

Si tu nombre de usuario de GitHub es “giraffeface”, entonces nómbralogiraffeface.github.io

Asegúrate de dejarlo configurado como “Público”. Si lo configuras como “Privado”, tu sitio web no se publicará.

Ahora puede hacer clic en “Crear repositorio a partir de una plantilla”.

¡Felicidades! ¡ Acabas de crear un blog SQL! (Más o menos). Es decir, es la plantilla predeterminada. Todavía tenemos que añadir cosas como tu nombre, quizás una breve biografía, tus nombres de usuario en redes sociales y, ya sabes… algunas entradas. Pero aparte de eso… ya tienes un blog publicado y visible en internet.

Puede que haya un pequeño retraso, pero GitHub finalmente reconocerá que has creado un repositorio de GitHub Pages, lo detectará automáticamente y creará tu sitio web. Y lo hará cada vez que realices un cambio.

Si han pasado unos minutos… ¡visita tu nuevo sitio! La página de inicio de tu blog simplemente mostrará el nombre del repositorio.

Así: https://chadblogtest.github.io

Personalización

Es genial que ahora tengas este blog tan elegante... pero probablemente no quieras que tu sitio diga "Nombre de autor predeterminado" por todas partes, y probablemente quieras cambiar el nombre del blog a algo que no sea "Nombre de mi blog".

Para editar estas cosas, tendremos que hacer algunos cambios en este archivo:_config.yml

Si no quieres proporcionar un correo electrónico o ninguno de los nombres de usuario de redes sociales, simplemente puedes comentarlos usando#

A continuación, podrás personalizar tu página de inicio. Aquí puedes escribir una breve biografía o un resumen del contenido de tu blog. Tú decides.

Para editar la página de inicio, el proceso es el mismo que seguimos para editar el _config.ymlarchivo, solo que esta vez, deberá editar el index.mdarchivo. Esta es su página de inicio.

Abre el programa index.md, pulsa el botón de edición y empieza a escribir. Si sabes usar Markdown, puedes usarlo para dar formato, pero si no, no te preocupes, escribe lo que quieras, sin preocuparte por HTML, CSS ni nada parecido.

Quizás tengas la tentación de ir inmediatamente a tu sitio para ver los cambios, pero como mencioné antes, debes esperar uno o dos minutos para que GitHub detecte que has realizado cambios y luego reconstruya tu sitio. Pero, finalmente, verás los cambios.

Tu primera entrada de blog

Quizás ya lo hayas notado, o quizás no, pero ya hay una entrada de blog ahí. Tómate un minuto para verla en el navegador, o incluso en tu teléfono para ver cómo se ve.

Luego, regresa a GitHub para ver los archivos. Las entradas del blog se almacenan en la _postscarpeta y allí encontrarás un archivo con el ejemplo de entrada.

Antes de profundizar demasiado en esto, vamos a repasar algunas de las cosas que necesitas saber sobre cómo crear un blog con GitHub Pages:

  • GitHub Pages utiliza un software llamado Jekyll para gestionar todos los aspectos técnicos. Jekyll convierte los archivos de tus entradas de blog en HTML con un formato legible que puedes visualizar en el navegador. Jekyll es capaz de hacer muchísimas cosas, pero por ahora, eso es todo lo que necesitas saber.

  • Los archivos de las entradas del blog siempre deben nombrarse utilizando este formato:yyyy-mm-dd-your-blog-post-name.md

  • Una publicación escrita el 28 de enero de 2021 se llamaría 2021-01-28-my-blog-post.md. Tenga cuidado de no usar fechas futuras porque la publicación no aparecerá.

  • Las publicaciones están escritas en Markdown. Si no lo conoces, no te preocupes, es muy sencillo. Consulta esta publicación de GitHub para aprender lo básico.

Eso es prácticamente todo lo que necesitas saber para empezar.

Vamos a crear una nueva entrada de blog.

  1. Navega a la _postscarpeta en GitHub.
  2. Haga clic Add file>Create new file
  3. Nombra tu archivo2026-07-06-your-new-blog-post.md
  4. Establece el título de tu entrada de blog usando un encabezado Markdown.
    • Escribe esto como la primera línea## This is my first blog post
  5. Añade contenido... escribe cosas al azar, lo que quieras.

  6. Inserta un bloque de código (los bloques de código se crean rodeando el fragmento de código con tres acentos graves en cada extremo y una "pista de idioma" opcional), copia y pega esto:

     ```tsql
 SELECT *
 FROM sys.tables
 WHERE [name] = 'SomeTable'
 ```
    • Nota importante : si utiliza código T-SQL, asegúrese de usar la tsqletiqueta. Esto le indicará a su sitio que desea usar el formato de estilo SSMS.
  7. Haz clic en la Previewpestaña para ver cómo va quedando hasta ahora.

Aquí tenéis una reproducción completa de cómo quedaría:

Una vez que guardes (confirmes) el archivo, después de uno o dos minutos tu nueva publicación aparecerá en la página principal, lista para que todos la lean.

Nota: Una vez que hayas resuelto esto, probablemente querrás eliminar esas entradas de blog de ejemplo. Puedes eliminarlas usando el botón de la papelera que está junto al botón de edición que has estado usando para editar archivos.

Revisando todos los cambios nuevos

Ahora que has actualizado la configuración, has cambiado el nombre de tu blog, personalizado la página de inicio y creado tu primera entrada... ahora puedes ir a comprobarlo todo...

¿Qué hacer ahora?

Si lo único que te interesa es publicar contenido, no necesitas saber mucho más. Una vez que tengas tu sitio web configurado, solo te quedará crear los archivos de las entradas del blog (nombrándolos correctamente), escribir el contenido y, tal vez, aprender un poco de Markdown.

Si te interesa profundizar en Jekyll, su funcionamiento interno u otras posibilidades, avísame; me encantaría escribir algunas entradas en el blog al respecto. Me he divertido mucho descubriendo cómo configurarlo y ponerlo en marcha.

Puedes añadirle muchas funciones interesantes a esta plantilla de forma bastante sencilla. Por ejemplo, una sección de comentarios en tus publicaciones, un tema oscuro para todo el sitio o páginas personalizadas, como mi página de seguimiento de libros o mi currículum . Las posibilidades son infinitas… pero tiene un precio: todo debe hacerse manualmente escribiendo o copiando y pegando código, aprendiendo nuevos lenguajes como HTML, CSS, Liquid, YAML… Aquí es donde plataformas como WordPress empiezan a destacar.

Entiendo que esta configuración en particular puede no funcionar para todos, y a la larga, puede incluso ser más complicada que simplemente usar una plataforma de blogs popular, con sus sofisticadas interfaces de usuario, complementos, etc. Pero me he divertido mucho aprendiendo sobre GitHub Pages, Jekyll, Markdown, etc., así que pensé en compartir cómo empecé e intentar facilitarte las cosas si esta es la opción que estás considerando.
